# 🚆 Railway CCTV Monitoring Dashboard

## Project Overview

This project is a prototype backend and frontend system for an **IoT-based Train CCTV Monitoring Dashboard**.
It allows railway operators to monitor CCTV video footage from train cameras, view stored video metadata, and analyze monitoring statistics through a dashboard interface.

The backend APIs are built using **FastAPI**, with **PostgreSQL** used as the relational database.
A lightweight frontend dashboard is integrated with the backend to visualize the data.

---

# 🎯 Features

### Dashboard Monitoring

* View **total videos monitored today**

* List CCTV video records

# 📡 API Endpoints

## 1️⃣ Get Videos List

GET /api/videos

Optional filters:

train_id
camera_id
from_date
to_date

Example:

/api/videos?train_id=1

---

## 2️⃣ Get Video Details

GET /api/videos/{video_id}

Example:

/api/videos/1

Returns detailed metadata about a specific video.

---

## 3️⃣ Dashboard Summary

GET /api/dashboard/summary

Returns system monitoring statistics:

* total_videos_today
* total_trains_monitored
* alerts_generated
* storage_usage

---

# 🗄 Database Schema

The system uses a **PostgreSQL relational database**.

### Tables

* trains
* cameras
* videos
* ai_detections
* alerts

### Relationships

* A **train** can have multiple cameras
* A **camera** belongs to a train
* A **video** belongs to a camera
* **AI detections** are linked to videos
* **Alerts** are generated from AI detections

# 📈 Architecture Improvements

The following improvements are proposed for production deployment:

### Data Flow

Introduce message queues (Kafka or RabbitMQ) for asynchronous video ingestion.

### Storage Strategy

Use AWS S3 for storing video files and move older videos to cold storage such as AWS Glacier.

### Scalability

Deploy backend using Docker containers and use load balancers for horizontal scaling.

### Performance Optimization

* Redis caching for dashboard metrics
* Pagination for large video lists
* Database indexing for faster queries

### Monitoring & Logging

Use structured logging and monitoring tools such as:

* ELK Stack
* Prometheus
* Grafana
